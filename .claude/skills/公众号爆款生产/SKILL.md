# 公众号爆款生产流水线

> 从热点链接到公众号草稿箱的一站式解决方案

## 🎯 功能概述

这是一个**端到端的自动化内容生产流水线**，将以下环节串联起来：

```
热点链接 → 选题推荐（含历史案例参考） → 用户选择 → 智能框架匹配 → 用户确认结构 → 文案生成（含个人素材注入） → 事实核查 → 标题生成 → 排版优化 → 敏感词检测 → 用户确认 → 发布草稿 → 存档+反馈闭环
```

## 📥 输入参数

| 参数 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| `热点链接` | ✅ 是 | 热点新闻链接，支持财联社/华尔街见闻/微博/知乎/贴吧/今日头条等 | - |
| `内容类型` | ❌ 否 | 教知识/聊观点/讲故事/晒过程 | 智能判断 |
| `执行模式` | ❌ 否 | 自动模式/交互模式 | 询问用户选择 |
| `目标平台` | ❌ 否 | 公众号/小红书/视频号 | 公众号 |

## ⚡ 双模式说明

### 模式一：🚀 自动模式（推荐）

全程自动执行，只在两个关键节点暂停：

```
自动抓取热点 → 自动选题包装（选TOP1） → 自动判断类型
→ 自动匹配框架+开篇 → 自动生成文案
→ ⏸️ 用户确认文案 → 自动下游流水线（标题→核查→敏感词→排版）
→ ⏸️ 用户确认最终成品
```

自动模式的默认行为：
- 选题选择：自动选择爆款概率最高的TOP 1选题
- 类型判断：AI自动判断最适合的内容类型
- 框架匹配：自动选择适配度最高的框架和最优开篇
- 下游流水线：标题→核查→敏感词→排版，全部自动执行

检查点只有 2 个：
| 检查点 | 位置 | 等什么 |
|--------|------|--------|
| ⏸️ 1 | 完整文案生成后 | 用户确认文案内容 |
| ⏸️ 2 | 最终成品（排版后）| 用户确认发布 |

### 模式二：🔧 交互模式

每步暂停，用户全程参与决策：

```
热点分析 → 选题包装 → ⏸️ 用户选择选题
→ 框架匹配+开篇 → ⏸️ 用户确认结构 → 文案生成
→ ⏸️ 用户审核文案 → 下游流水线 → ⏸️ 用户确认发布
```

检查点 4 个：
| 检查点 | 位置 | 等什么 |
|--------|------|--------|
| ⏸️ 1 | 选题包装后 | 用户选择选题方向 |
| ⏸️ 2 | 框架+开篇匹配后 | 用户确认结构和开篇 |
| ⏸️ 3 | 完整文案生成后 | 用户审核文案 |
| ⏸️ 4 | 最终成品（排版后）| 用户确认发布 |

## 🔄 完整工作流程

### ⚡ Phase 1: 热点解析 + 选题推荐（连续执行，不可中断）

> **重要**：Step 1 和 Step 2 必须在**同一个回复**中完成，不可中途暂停等待用户。

**执行步骤**：
1. 使用 `mcp__web_reader__webReader` 工具抓取所有热点链接内容
2. 同时读取以下知识库（并行执行）：
   - `09.经验沉淀/爆款方法论/13大爆款元素.md`
   - `01.用户画像/我的用户画像.md`
   - **`09.经验沉淀/爆款方法论/数字生命卡兹克-选题特点萃取.md`** ⚠️ 必读
   - **`03.选题决策/选题模板库.md`** ⚠️ 必读：套用验证过的选题公式
   - **`08.数据反馈/失败案例库.md`** ⚠️ 必读：避免踩同样的坑
   - 按热点相关的内容类型，读取对应的爆款案例库（选1-2个）：
     - `08.数据反馈/爆款案例库/教知识类爆款.md`
     - `08.数据反馈/爆款案例库/聊观点类爆款.md`
     - `08.数据反馈/爆款案例库/讲故事类爆款.md`
     - `08.数据反馈/爆款案例库/晒过程类爆款.md`
3. 提取热点核心信息
4. **数据可验证等级标注**：对每条关键数据标注来源可信度等级，供Step 5写文案和Step 6事实核查参考：
   | 等级 | 来源类型 | 文案处理方式 |
   |------|---------|------------|
   | A级 | 权威媒体报道/官方公告/论文 | 可直接引用具体数字 |
   | B级 | 行业媒体/知名分析师/企业官方社交媒体 | 可引用但建议标注来源 |
   | C级 | 社交媒体推文/个人博客/未经验证的转发 | 必须模糊化处理（如"据报道"/"有消息称"），或在Step 6核查后决定是否保留 |
5. 匹配13大爆款元素，结合卡兹克选题特点萃取中的12种切入角度，参考选题模板库中验证过的公式，生成5个选题并打分
6. **选题排除检查**：对照失败案例库，排除与历史失败案例相似的选题方向（如选题过于宽泛、缺乏具体痛点等）
6. **一次性输出**：热点摘要 + 5个爆款选题

**目标**：全网全方位深度搜集信息，生成深度分析报告

**执行方式**：
1. 将用户选择的选题 + 热点链接作为输入，直接调用「热点追踪」skill

**调用后检查**：

### Step 10.5: 自动生成配图/截图说明

**目标**：根据文案类型自动生成配图说明，并写入存档文件

**配图类型规则（强制）**：

| 文案来源 | 配图类型 | 格式 |
|---------|---------|------|
| 热点类（线路1：热点追爆款） | 真实信源截图 | `【截图①：{原文链接} ，{截图内容描述}】` |
| 教知识/干货类 | 设计图 | `【配图①：{图片类型}，{具体描述内容}】` |
| 讲故事/晒过程类 | 实拍图/场景图 | `【配图①：{图片类型}，{具体描述内容}】` |

**热点类截图说明生成规则**：
1. 从Phase 1用户提供的热点链接中，选取与文案正文直接相关的链接
2. 每个截图标注：原文链接URL + 截图内容描述（一句话说明截图展示什么）
3. 标注截图在正文中的建议插入位置（在哪个段落之后）
4. 截图数量：与正文引用的信源数量一致，通常3-6张

**热点类录屏标记规则**：
> 录屏 ≠ 截图。截图是静态画面，录屏是滚动录制页面的动态过程，适合展示推文上下文、评论区反应、时间线等需要"滑动才能看全"的内容。

1. **判断是否需要录屏**：以下场景必须标记录屏（而非仅截图）：
   - 推文内容较长，需要滚动才能看完正文
   - 需要展示评论区的热烈反应/争议讨论
   - 需要展示发布时间、互动量（点赞/转发/评论数）等动态信息
   - 需要展示同一账号的多条相关推文时间线
2. **录屏标记格式**：`【录屏①：{原文链接} ，{滚动录制描述}】`
3. **滚动录制描述要求**：必须具体说明录制路径，包含：
   - 起始位置（如"从推文正文顶部"）
   - 滚动方向和终止位置（如"往下滑到评论区前几条"）
   - 重点展示内容（如"展示发布时间和互动量"）
4. **录屏与截图的关系**：同一信源可以同时有截图（静态关键画面）+ 录屏（动态滚动过程），录屏标记放在对应截图下方
5. **录屏数量**：热点文章通常3-5条，每条对应一个关键信源

**录屏标记示例**：
```
【录屏①：https://x.com/elonmusk/status/xxx ，滚动录制马斯克发布XX的推文，从推文正文往下滑到评论区前几条，展示发布时间和互动量】
【录屏②：https://x.com/sama/status/xxx ，滚动录制Sam Altman的推文，展示正文内容和关键数据，往下滑到评论区展示用户反应】
```

**热点类动画录屏（动录）标记规则**：
> 动录 ≠ 录屏。录屏是手动滚动页面的过程，动录是录制页面自身的动画效果、自动播放视频、GIF动图、交互演示等动态内容，不需要手动滚动。

1. **判断是否需要动录**：以下场景必须标记动录（而非截图或录屏）：
   - 信源页面包含自动播放的视频/Demo演示
   - 信源页面有动态图表、动画效果、交互演示
   - 推文附带的视频内容是关键信息（如产品发布视频、功能演示）
   - 需要展示AI工具的实际运行效果/操作过程
2. **动录标记格式**：`【动录①：{原文链接} ，{动画内容描述}】`
3. **动画内容描述要求**：必须具体说明：
   - 录制什么动态内容（如"录制推文附带的Grok 4.20功能演示视频"）
   - 录制时长建议（如"录制前15秒核心演示部分"）
   - 重点捕捉画面（如"重点捕捉代码自动生成的过程"）
4. **动录与截图/录屏的关系**：同一信源可以同时有截图（静态）+ 录屏（滚动）+ 动录（动画），三者互补不冲突
5. **动录数量**：视信源中动态内容数量而定，通常1-3条

**动录标记示例**：
```
【动录①：https://x.com/elonmusk/status/xxx ，录制推文附带的Grok 4.20功能演示视频，录制前15秒核心演示部分，重点捕捉多模态交互效果】
【动录②：https://x.com/OpenAI/status/xxx ，录制GPT-5.3 Codex的代码生成演示动画，重点捕捉实时代码补全的过程】
```

**输出格式（热点类）**：
```
【截图①：{URL} ，{内容描述}】→ 建议位置：{段落描述}之后
【录屏①：{URL} ，{滚动录制描述}】→ 建议位置：对应截图下方
【动录①：{URL} ，{动画内容描述}】→ 建议位置：对应截图下方
【截图②：{URL} ，{内容描述}】→ 建议位置：{段落描述}之后
【录屏②：{URL} ，{滚动录制描述}】→ 建议位置：对应截图下方
...
```

**⚠️ 此步骤在用户确认发布后自动执行，不需要用户额外确认。截图说明必须写入Step 11的存档文件中。**
