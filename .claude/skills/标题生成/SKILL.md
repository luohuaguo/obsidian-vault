## 🚀 快速启动模式（默认）

当用户提供简单需求时：
1. 立即根据现有信息开始工作
2. 不强制要求用户提供完整信息
3. 快速给出结果
4. 然后询问是否需要优化

只有当用户明确要求"详细分析"或"深度优化"时，才启动完整流程。

## 📋 标题生成规则

以下是用户的视频/文案内容，生成20个标题，要求如下：

① 标题禁止像新闻稿，它得有情绪，有直接和用户相关的利益点，应该是一个能够吸引人点击的视频标题。

② 标题要像写畅销书标题一样思考，要形成更浓缩更有冲击力的短语

③ 必须要有爆款元素在里面（最好是对比/对立元素），字数尽量在13-20个字。开始吧

④ 标题生成后，结尾补充选题建议，根据视频号、抖音、B站、小红书、公众号这些平台的调性,推荐能够吸引用户点击的标题。

## 🔗 被上游skill调用时的规则

**⚠️ 重要：本skill是标题生成的唯一入口，所有需要标题的场景都必须通过调用本skill完成，禁止手动生成或用后台agent替代。**

当被以下上游skill调用时：
- 对话转文案
- 公众号爆款生产
- 内容创作工作流
- 实战复盘转内容
- 灵感碎片转文案

**协作规则：**
1. 接收上游传入的完整文案内容
2. 读取知识库后生成20个标题
3. 交互模式：展示给用户选择 → 返回用户选定的标题
4. 自动模式：直接返回TOP 1推荐标题
5. 标题确定后，上游skill负责写入文件frontmatter的title字段
