# 爆款拆解改写

> 保留市场验证过的骨架，换上你自己的血肉

## 🎯 功能概述

这是一个**从别人的爆款到自己的原创**的改写 skill，解决一个核心问题：
你看到一篇别人的爆款，想用它的"骨架"写一篇自己的文章，但不知道怎么拆、怎么改、怎么避免抄袭。

```
别人的爆款全文 → 深度拆解 → 骨架提炼 → 选择改写策略 → 素材替换 → 文案生成 → 原创度自检 → 下游流水线
```

与其他 skill 的区别：

| Skill | 素材来源 | 核心动作 |
|-------|---------|---------|
| 公众号爆款生产 | 外部热点链接 | 从热点出发，自己选题自己写 |
| 灵感碎片转文案 | 脑子里的想法 | 从碎片出发，AI判断类型再写 |
| 实战复盘转内容 | 自己的实战过程 | 从经历出发，提炼方法论再写 |
| **爆款拆解改写** | **别人的爆款文案** | **从别人的成品出发，拆骨架→换血肉** |

## 📥 输入参数

| 参数 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| `爆款文案全文` | ✅ 是 | 粘贴别人的完整爆款文案 | - |
| `执行模式` | ❌ 否 | 自动模式/交互模式 | 询问用户选择 |
| `改写策略` | ❌ 否 | S/A/B/C 四种策略 | 自动模式默认S级，交互模式由用户选择 |
| `自己的选题` | ❌ 否 | 想改写成什么选题（策略A/B/C需要） | 策略S不需要 |
| `目标平台` | ❌ 否 | 公众号/小红书/视频号 | 公众号 |

## ⚡ 双模式说明

接收到爆款文案后，先询问用户选择执行模式（如果用户已指定则跳过）：

### 模式一：🚀 自动模式（推荐）

全程自动执行，只在文案生成后暂停一次让用户确认：

```
拆解 → 自动提炼骨架 → 默认S级策略 → 自动从个人自传/社群总结提取素材
→ 自动生成文案 → ⏸️ 用户确认文案 → 自动原创度自检 → 自动下游流水线
→ ⏸️ 用户确认最终成品 → 发布
```

自动模式的默认行为：
- 改写策略：默认 S级（最小改动改写）
- 新公式/新结构：自动追加到对应模板库（不询问）
- 素材来源：自动从个人自传 + 社群总结提取，缺口素材合理推理
- 原创度自检：自动执行，未通过则自动优化直到通过
- 下游流水线：标题→核查→敏感词→排版，全部自动执行

检查点只有 2 个：
| 检查点 | 位置 | 等什么 |
|--------|------|--------|
| ⏸️ 1 | 完整文案生成后 | 用户确认文案内容 |
| ⏸️ 2 | 最终成品（排版后）| 用户确认发布 |

### 模式二：🔧 交互模式

每步暂停，用户全程参与决策：

```
拆解 → ⏸️ 确认 → 选策略 → ⏸️ 确认 → 素材替换 → ⏸️ 确认大纲
→ 文案生成 → ⏸️ 确认 → 原创度自检 → ⏸️ 确认 → 下游流水线 → ⏸️ 确认发布
```

检查点有 6 个：
| 检查点 | 位置 | 等什么 |
|--------|------|--------|
| ⏸️ 1 | Step 1 拆解完 | 确认拆解结果，是否追加新公式 |
| ⏸️ 2 | Step 2 策略展示后 | 选择 S/A/B/C 策略 |
| ⏸️ 3 | Step 3 改写大纲输出后 | 确认大纲、补充缺口素材 |
| ⏸️ 4 | Step 4 完整文案输出后 | 确认文案内容 |
| ⏸️ 5 | Step 5 自检报告输出后 | 确认原创度 |
| ⏸️ 6 | 最终成品（排版后）| 确认发布 |

## 🔄 完整工作流程

### Step 1：爆款输入 + 深度拆解

接收用户粘贴的爆款文案全文后，执行 5 层深度拆解。

**必读知识库**（并行读取）：
- `09.经验沉淀/爆款方法论/13大爆款元素.md`
- `02.文案结构/00.框架使用说明书.md`
- `09.经验沉淀/爆款方法论/数字生命卡兹克-文案规律萃取.md`
- 按判断出的内容类型，读取对应的开篇模板库

**5 层拆解：**

#### 第1层：内容类型判断

判断这篇爆款属于哪种内容类型：
- 教知识（方法论拆解/追求更好/解决问题/降低成本/丰富谈资）
- 聊观点（行业导向/人群导向/热点借势/对话采访/创业感悟）
- 讲故事（小有成就/成功案例/平凡英雄/猎奇苦难/共鸣苦难）
- 晒过程（成果揭秘/经验总结）

#### 第2层：结构骨架拆解

```
开篇钩子：
  - 原文开篇（逐字摘录前3-5句）
  - 开篇类型：痛点直击/故事引入/金句观点/数据冲击/强势判断/反常识
  - 对应开篇公式库中的哪个公式？（如果是新公式，标注"⚡新发现"）
  - 开篇句式：否定句/肯定句/疑问句

主体结构：
  - 总共几层论证/几个段落
  - 每层的功能：论证方向 × 论证方式
  - 段落之间的递进关系：并列/递进/对比/转折
  - 论证结构：归纳论证/反证论证

结尾策略：
  - 结尾类型：重申升华/行动号召/情感留白/社群导流/立Flag
  - 是否有金句（逐字摘录）
  - 结尾钩子形式
```

#### 第3层：爆款元素识别

对照13大爆款元素逐一检查，标注命中了哪些：

```
生存层：□成本/金钱 □最差/避坑 □荷尔蒙
发展层：□人群标签 □场景 □数据/数字 □对比/对立
超越层：□反认知 □怀旧 □猎奇/揭秘 □头牌/权威 □高维认知 □钩子问题

命中数量：X/13
核心爆款元素（最关键的2-3个）：XXX
```

#### 第4层：情绪节奏图

标注全文的情绪起伏：

```
开篇 → [情绪状态] → 第一层 → [情绪状态] → 第二层 → [情绪状态] → ... → 结尾 → [情绪状态]

节奏模式：平起高收 / 高开高收 / 低开高收 / 波浪递进
情绪转折点：在第X段，从[情绪A]转向[情绪B]
```

#### 第5层：数据特征

```
总字数：XXX字
推荐时长：X-X分钟
段落数：X段
平均段落长度：X行
金句密度：每X字一句
"我"视角占比：X%
口语词密度：高/中/低
对话感：强/中/弱
```

#### 🔄 新公式/新结构发现

拆解完成后，自动检查：
- 开篇公式是否在现有开篇模板库中已有？
  - 已有 → 标注对应公式名
  - 未有 → 标注"⚡新发现"，提示用户是否要追加到开篇模板库
- 结构模式是否在现有框架中已有？
  - 已有 → 标注对应框架名
  - 未有 → 标注"⚡新发现"，提示用户是否要追加到结构文档

**输出格式：完整的拆解报告**

**模式分流：**
- 🚀 自动模式：输出拆解报告后，自动追加新公式/新结构到对应模板库，直接进入 Step 2
- 🔧 交互模式：⏸️ 等待用户确认拆解结果，确认是否追加新公式，再进入下一步

---

### Step 2：骨架提炼 + 改写策略选择

基于拆解结果，提炼可复用骨架，并推荐改写策略。

#### 2.1 可复用骨架输出

```
┌─ 可复用骨架 ──────────────────────────┐
│                                       │
│ 内容类型：教知识/聊观点/讲故事/晒过程    │
│ 开篇公式：[公式名] + [原文示例]         │
│ 主体结构：[框架名] + [层数] + [论证方式] │
│ 结尾策略：[策略] + [金句(如有)]         │
│ 爆款元素：[命中元素列表]                │
│ 情绪节奏：[节奏模式]                   │
│ 推荐字数：[区间]                       │
│                                       │
└───────────────────────────────────────┘
```

> 此骨架格式与「选题匹配爆款模板」的输出格式统一，下游文案 skill 可无缝消费。

#### 2.2 四种改写策略

根据拆解结果，推荐最适合的策略，并说明理由：

**S级：最小改动改写（同话题换人）**
- 开篇：基本不动，只换品牌词/产品名等必要关键词
- 骨架：不动
- 知识点：不动（同领域同话题）
- 替换内容：
  - 作者个人经历 → 你的真实经历（从个人自传提取）
  - 作者案例/数据 → 你自己验证过的案例/数据
  - 结尾钩子 → 你的引流钩子（从引流钩子模板提取）
- 表达口吻 → 用你的风格重写（调用公众号文案风格萃取）
- 适用场景：同赛道、同话题，你有类似经历可以替换
- ⚠️ 原创度要求：最高（必须通过原创度自检）

**A级：同类型换题（同结构换题）**
- 开篇：保留公式结构，替换关键词换成你的选题
- 骨架：保留论证结构和节奏
- 知识点：换成你的选题
- 替换内容：所有素材、案例、论据全部换
- 适用场景：同类型但不同话题

**B级：跨类型迁移**
- 保留情绪节奏和段落节奏
- 用目标类型的框架重新组织结构
- 适用场景：原文是聊观点，你想改成教知识

**C级：元素重组**
- 只保留爆款元素组合，结构和选题都重新设计
- 适用场景：只觉得这篇的元素组合很妙，想借鉴思路

#### 2.3 策略推荐逻辑

```
你和原作者在同一赛道，且有类似经历？
  ├─ 是 → 推荐 S级（最小改动改写）
  └─ 否 → 你想写同类型内容？
           ├─ 是，同类型不同选题 → 推荐 A级
           ├─ 是，但想换类型 → 推荐 B级
           └─ 只想借鉴元素组合 → 推荐 C级
```

**模式分流：**
- 🚀 自动模式：自动选择 S级策略，跳过用户选择，直接进入 Step 3
- 🔧 交互模式：⏸️ 展示四种策略的说明和推荐，等待用户选择

---

### Step 3：素材替换

根据用户选择的策略，收集替换素材。

**必读知识库**（并行读取）：
- `04.我的上下文/我的个人上下文/个人自传（持续更新）.md`
- `04.我的上下文/我的个人专业思考/社群总结（持续更新）.md`
- `04.我的上下文/我的个人上下文/公众号结尾引流钩子模板.md`

#### 策略S：最小改动改写

逐段对照原文，标注需要替换的内容：

```
原文第X段（保留/替换）：
  原文：[原文内容]
  替换为：[从个人自传/社群总结中提取的对应素材]
  替换理由：个人经历/案例数据/结尾钩子
```

需要用户补充的素材（如果个人自传中没有对应经历）：
- ❓ 第X段需要：[具体描述需要什么素材]
- ❓ 第X段需要：[具体描述需要什么素材]

#### 策略A：同类型换题

用户需要提供：
1. 你的选题是什么？
2. 你有哪些相关素材/案例/数据？

AI自动从知识库补充：
- 从个人自传提取相关经历
- 从社群总结提取相关观点

输出「改写大纲」：
```
开篇：[原文开篇公式] → 替换关键词为 [你的选题关键词]
第一层：[原文论证方向×方式] → 换成 [你的素材]
第二层：[原文论证方向×方式] → 换成 [你的素材]
第三层：（如有）→ 换成 [你的素材]
结尾：[原文结尾策略] → 换成 [你的引流钩子]
```

#### 策略B：跨类型迁移

用户需要提供：
1. 目标内容类型（教知识/聊观点/讲故事/晒过程）
2. 你的选题

AI自动匹配：
- 从框架使用说明书中选择目标类型的最优框架
- 保留原文的情绪节奏曲线
- 用目标类型的结构重新组织

#### 策略C：元素重组

用户需要提供：
1. 想保留哪些爆款元素
2. 你的选题方向

AI围绕选定元素重新设计结构。

**模式分流：**
- 🚀 自动模式：自动从个人自传+社群总结提取素材，缺口素材合理推理，跳过确认直接进入 Step 4
- 🔧 交互模式：⏸️ 输出「改写大纲」，等待用户确认、补充缺口素材

### Step 4：文案生成

**必读知识库**（并行读取）：
- `04.我的上下文/我的个人上下文/公众号文案风格萃取.md` ⚠️ 必读
- `04.我的上下文/AI创作规范.md`

#### 4.1 开篇处理（核心差异点）

**策略S/A：开篇作为硬模板**
- 直接以原文开篇为模板
- 只替换必要的关键词（品牌词/产品名/选题关键词）
- 保留原文的句式、节奏、语气
- 不走文案 skill 自己的开篇设计流程

```
原文开篇：[逐字摘录]
      ↓ 只替换关键词
改写开篇：[替换后的版本]
```

**策略B/C：开篇作为软参考**
- 参考原文开篇的公式和情绪
- 但用目标类型的开篇库重新设计
- 保留原文的情绪强度

#### 4.2 主体生成

根据改写策略和确认的大纲，生成完整文案：

**风格约束**（所有策略通用）：
- 必须使用用户的口吻（参照公众号文案风格萃取）
- 高频口语词：那、就、其实、所以、你会发现、呢、然后
- 追问式论证："那为什么？因为..."、"那具体怎么做呢？"
- 短段落原则：每段不超过3-4行
- 禁用AI味词汇：综上所述、建议您、首先其次最后、通过可以发现
- 数字具体化：用精确数字代替模糊描述
- 场景还原：用画面感代替概括叙述

**策略S特殊规则**：
- 保留原文的段落节奏和结构
- 只替换标注为"替换"的段落内容
- 保留段落之间的过渡方式
- 用用户的口吻重写每一句（即使是保留的段落，也要过一遍口吻转换）

**策略A特殊规则**：
- 保留原文的论证结构
- 开篇只换关键词
- 每层论证用你的素材重新填充
- 保留原文的情绪节奏

**策略B/C特殊规则**：
- 路由到对应的文案 skill，传入骨架作为软参考
- 教知识 → 调用「教知识文案」skill 的创作流程
- 聊观点 → 调用「聊观点文案」skill 的创作流程
- 讲故事 → 调用「讲故事文案」skill 的创作流程
- 晒过程 → 调用「公众号爆款生产」skill 的创作流程

#### 4.3 结尾处理

所有策略统一：
- 结尾钩子替换为你自己的（从引流钩子模板提取）

- 如果原文金句不适用，不硬写

**模式分流：**

### Step 5：原创度自检

对生成的文案进行 5 项原创度检查，确保避免抄袭：

#### 5.1 五项检查

| 检查项 | 标准 | 通过条件 |
|--------|------|----------|
| ① 开篇相似度 | 允许高度相似（S/A级） | 只换了必要关键词即可 ✅ |
| ② 个人经历段 | 必须完全不同 | 用的是你的真实经历 ✅ |
| ③ 案例/数据 | 必须替换 | 用的是你自己的案例数据 ✅ |
| ④ 表达口吻 | 必须是用户风格 | 符合公众号文案风格萃取 ✅ |
| ⑤ 连续相似度 | 任意连续3句不能与原文高度相似 | 逐段对比检查 ✅ |

#### 5.2 逐段对比报告

```
段落1（开篇）：
  原文：[原文前3句]
  改写：[改写后前3句]
  相似度：高/中/低
  判定：✅ 开篇允许高相似 / ❌ 需要进一步改写

段落2：
  原文：[原文内容]
  改写：[改写后内容]
  相似度：高/中/低
  判定：✅ 通过 / ❌ 连续3句以上高度相似，需要重写

...（逐段检查）
```

#### 5.3 总体判定

```
原创度评分：X/100
  - 开篇：✅（允许相似）
  - 个人经历：✅/❌
  - 案例数据：✅/❌
  - 表达口吻：✅/❌
  - 连续相似度：✅/❌

总体判定：✅ 通过原创度自检 / ❌ 需要优化以下段落：[具体段落]
```

如果未通过 → 自动优化标注的段落 → 重新检查 → 直到通过

### 署名测试（不可替代性检查）

> **核心问题：把署名换成别人，这篇文案还能成立吗？**

逐项检查：
- [ ] 文中是否有**只有用户才有的经历**？（个人实战、真实案例、踩过的坑）
- [ ] 文中是否有**独家数据或独家判断**？（不是公开信息的二次搬运）
- [ ] 去掉所有个人印记后，和通用AI生成的内容**区别在哪**？（必须能说清）

**判定：**
- ✅ 至少2项有明确答案 → 通过，继续下游流水线
- ❌ 少于2项 → **必须补入个人素材**（从 `04.我的上下文` 或 `思想力武器库` 中调取），重新过一遍自检

**模式分流：**

- 🔧 交互模式：⏸️ 输出原创度自检报告，等待用户确认

## ⚠️ 核心原则

### 改写的"度"

```
保留优先级（从高到低）：
  开篇钩子 > 情绪节奏 > 段落结构 > 论证方式 > 具体内容
  几乎不动   尽量保留    保留骨架    可微调     全部替换
```

### 绝对不能做的事

- ❌ 不能整段照搬原文（开篇除外，开篇允许高度相似）
- ❌ 不能编造个人经历（必须从个人自传中提取真实素材）
- ❌ 不能编造数据和案例（必须是你自己验证过的）
- ❌ 不能保留原作者的个人标签和身份信息
- ❌ 不能连续3句以上与原文高度相似

### 必须做到的事

- ✅ 开篇保留原文的钩子结构（这是市场验证过的）
- ✅ 所有个人经历必须是用户的真实经历
- ✅ 表达口吻必须是用户风格（参照公众号文案风格萃取）
- ✅ 结尾钩子必须替换为用户的引流钩子
- ✅ 最终必须通过原创度自检
