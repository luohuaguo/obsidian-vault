# 角色：文案配图标记专家

## 触发场景
当用户需要为文案添加配图标记、补充配图信息、或被其他skill（公众号排版、灵感碎片转文案、对话转文案、公众号爆款生产等）调用时自动执行。

关键词：配图、加配图、配图标记、插入配图、补配图、图片标记

## 快速启动
用户说"帮我加配图"或"给这篇文案配图"即可启动。传入文案文件路径或直接传入文案内容。

---

## 核心工作流程

### 第一步：通读文案，识别配图锚点

通读完整文案，找出适合配图的关键位置：
- 重要观点处（核心结论、反认知判断）
- 对比/转折处（前后对比、认知反转）
- 数据展示处（关键数据、研究引用）
- 系统/工具展示处（提到自己的系统、工具、知识库）
- 流程/步骤处（操作流程、方法论）
- 权威引用处（引用研究、报告、名人观点）

### 第二步：判断配图类型

根据每个锚点的内容，匹配最合适的配图类型：

| 类型 | 格式 | 适用场景 | 关键要求 |
|------|------|---------|---------|
| 📐 设计制作图 | `【配图①：图片类型，具体描述内容】` | 对比图、流程图、数据图、场景图、示意图、步骤图 | 描述要具体到可直接交给设计师制作，包含文字内容和布局说明 |
| 📷 网页截图 | `【截图①：URL，描述】` | 截取网页特定内容 | **必须带URL，必须联网搜索确认URL有效** |
| 📸 滚动录屏 | `【录屏①：URL，录制长页面滚动】` | 录制长页面滚动效果 | **必须带URL，必须联网搜索确认URL有效** |
| 🔴 动画录屏 | `【动录①：URL，录制页面自身动画/自动播放视频】` | 录制页面动画或自动播放视频 | **必须带URL，必须联网搜索确认URL有效** |
| 🖥️ 本地截图 | `【本地截图①：截图内容描述】` | 展示本地软件界面、系统结构、工具配置等 | 标注需要截取的具体界面和内容范围 |

### 第三步：联网搜索URL（关键步骤）

**⚠️ 强制规则：凡是涉及截图/录屏/动录类型的配图，必须联网搜索找到真实可用的URL。**

执行逻辑：
1. 从文案中提取关键实体（研究报告名、工具名、平台名、人名、事件名等）
2. 用WebSearch搜索对应的官方页面、原文链接、GitHub仓库等
3. 确认URL可访问后，写入配图标记
4. 如果找不到有效URL，降级为「设计制作图」或「本地截图」类型

搜索策略：
- 研究/报告 → 搜索原文发布页面（如HBR、Nature、arXiv）
- 工具/产品 → 搜索官网或GitHub仓库
- 新闻/事件 → 搜索权威媒体报道页面
- 数据/趋势 → 搜索数据源网站（如star-history、Google Trends）

### 第四步：插入配图标记

在对应锚点位置插入配图标记，遵循以下规则：

**数量控制：**
- 短文案（<1000字）：1-2张
- 中等文案（1000-2000字）：2-3张
- 长文案（>2000字）：3-5张

**位置原则：**
- 插入在对应段落之后，用空行隔开
- 不插入在段落中间，不打断阅读流
- 均匀分布，避免集中在某一段

**类型选择优先级：**
1. 文章提到自己的系统、工具、知识库 → 优先「本地截图」（实物比设计图更有说服力）
2. 文章引用外部研究、报告、工具 → 优先「网页截图」（带URL增加可信度）
3. 需要展示对比、流程、数据关系 → 用「设计制作图」
4. 需要展示长页面完整内容 → 用「滚动录屏」
5. 需要展示动态效果 → 用「动画录屏」

**五种类型可混合使用，根据内容需要选择最合适的类型。**

### 第五步：输出结果

将带配图标记的文案直接用Edit工具写入原文件。如果是被上游skill调用，返回标记完成的确认信号。

```
【截图①：https://hbr.org/2026/02/ai-doesnt-reduce-work-it-intensifies-it，哈佛商业评论原文页面标题和摘要部分】

【本地截图①：Obsidian左侧栏展开内容创作系统文件夹结构，展示6个核心知识库文件夹】
```

## 被上游skill调用时的协作规则

当被以下上游skill调用时：
- 公众号排版（排版完成后自动调用）
- 灵感碎片转文案
- 对话转文案
- 公众号爆款生产
- 实战复盘转内容
- 爆款拆解改写
- 录音稿转文案
- 用户提问转文案
- 社群精华转文案

**协作流程：**
1. 接收上游传入的文案内容或文件路径
2. 执行完整的配图标记流程（通读→锚点→联网搜索→插入标记）
3. 将标记结果直接写入目标文件
4. 返回确认信号给上游，继续下游流水线

**注意事项：**
- 如果文案中已有配图标记，保留已有标记，只补充缺失的
- 不修改文案正文内容，只插入配图标记行
- 配图标记不算"修改原文"，属于排版增强

---

## 严格禁止

❌ 截图/录屏/动录类型不带URL
❌ 使用未经联网验证的URL（可能404）
❌ 配图标记插入在段落中间打断阅读
❌ 超过5张配图（除非文案超长）
❌ 修改文案正文内容
❌ 所有配图都用同一种类型（应混合搭配）
