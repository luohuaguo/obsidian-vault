## 一、核心选题公式

公式：目标人群的常见错误认知 × 「不是A而是B」的认知翻转

| 变量             | 可选值示例                                                     |
|-----------------|---------------------------------------------------------------|
| 常见错误认知      | 讲干货就有流量 / 线下内容搬线上 / 短视频=给人收获感 / 复盘=看回放 / 有违禁词会封号 |
| 认知翻转（不是A而是B） | 短视频是打广告的 / 复盘是有颗粒度的 / 封号不是因为违禁词 / 努力不等于盲目重复 |

底层情绪驱动力：
- 恐惧（我是不是也在犯这个错）
- 好奇（那正确的到底是什么）
- 收藏欲（这个方法我要存下来）


## 二、开篇设计

优先级排序：警告式 > 痛点提问式 > 观点冲突式

### 方向A：颠覆认知（直接否定常识）

句式模板：
- 「短视频是打广告用的，不是给别人有收获感用的。每一条短视频就是一条广告。」
- 「这个平台从【时间】开始就已经没有【大家以为的概念】了。现在【真正的规则】叫什么你知道吗？」
- 「我就没有见过哪个讲知识讲干货的人是真的有流量的。你去他们直播间全都是稀稀拉拉的【小数字】人在线。」

### 方向B：反面案例引入

句式模板：
- 「看【头部A】、看【头部B】、看【头部C】，你会发现他们不都是【某个共同做法】吗？把【旧做法】搬到【新场景】，别人行凭什么我们不行呢？以前【某类人】在做【新事】的时候一定会有一个坏习惯。」
- 「之前从来没有这样【负面结果】过，从【高数字】掉到【低数字】。所以你现在能不能估算得出来是哪些原因？」

### 方向C：痛点提问

句式模板：
- 「原来从在线【大数字】人，现在流量下滑很厉害。那我们怎么去拓展自己新的思路？」
- 「拆解一下我们的【具体工作流】，从起始开始我们会把它分为【数字】个阶段。」


## 三、论证过程设计

### 推荐结构A：标准3论点（先破后立）

论点1（占全文约25%）：
→ 为什么大家都在犯这个错？
→ 找到错误认知的根源
→ 用具体案例/场景佐证
→ 小结论：所以这条路走不通

论点2（占全文约35%）：
→ 正确的做法到底是什么？
→ 给出方法论的核心原理
→ 用对比/类比强化说服力
→ 小结论：底层逻辑是这个

论点3（占全文约20%）：
→ 怎么具体操作？
→ 案例论证（举1-2个实操例子）
→ 回扣主题

### 推荐结构B：递进式4段（适合深度方法论，4条高数据爆款验证）

第1段·深入分析：
→ 用具体案例/场景
→ 深入分析原因/机制
→ 得出洞察

第2段·核心方法：
→ 给出核心方法/观点
→ 用对比/类比强化说服力

第3段·补充论证：
→ 补充论证或延伸
→ 回应可能的疑问
→ 升华主题

### 论据组合原则：
- 「错误做法」和「正确做法」必须成对出现
- 至少1个具体的操作步骤或可执行的方法
- 类比要用大众都能理解的事物（「有身份有案例，跟你上厕所要带纸一样」）
- 方法论要分层/分步，不要一锅端

### 可选论据方向：

| 论据方向   | 典型用法                                                 |
|-----------|-------------------------------------------------------- |
| 错误示范   | 「你认为客户缺的是怎么做——你觉得劝得动他吗？」               |
| 正确示范   | 「他不是刺激痛点，而是让客户自己感觉到痛了」                  |
| 分步拆解   | 「第一个叫0到100，第二个叫100到后面」                        |
| 类比说明   | 「就像一个胖的人，你跟她说怎么吃，你能改变她吗？」            |
| 头部案例   | 「16岁打工踩缝纫机，现在电商主播培训一姐，常年四五千人在线」    |


## 四、结尾强化

首选：总结归纳式（该类型下平均点赞13,396——最高）

句式模板：
- 「它底层一定是【本质定义】的工作，绝对不是大家过去以为的那些东西。」
- 「你必须得搬走客户到产生行为中间的【阻碍A】、【阻碍B】和【阻碍C】，搬走这些他才会【期望行动】。」
- 「所以你告诉客户一个他本来就知道的道理，是不会让他产生行为的。」

备选：观点收束式

句式模板：
- 「接下来呢想【目标】还得把自己的【具体事项】做起来。所以今天梳理完之后，回去又得调整。」


## 五、选题生成技巧

快速批量生成法：

第一步：列出目标用户最常做的10件事
第二步：逐一检验——这件事的「常见做法」是否=「正确做法」？
第三步：找到「常见做法≠正确做法」的那几件
第四步：包装为「不是A而是B」的选题

示例：
- 常见做法「讲干货吸引粉丝」→ 正确做法「短视频是打广告的」
- 常见做法「复盘就是看回放」→ 正确做法「复盘是有颗粒度和方法论的」
- 常见做法「避开违禁词防封号」→ 正确做法「封号的底层逻辑不是违禁词」
- 常见做法「线下课搬线上」→ 正确做法「线上线下是两套逻辑」
- 常见做法「努力重复练习」→ 正确做法「刻意练习而非盲目重复」
