## 四、AI工具实战经验

### 实际应用场景：写生产日报

**以前的痛点**
- 每天花30-60分钟整理日报（产量、能耗、良品率、设备故障、改进建议）
- 最烦"描述问题和分析原因"这一栏，写来写去就那几句话

**现在的做法**
- 用OpenClaw（飞书AI助手），把数据和要点丢给它
- 1分钟生成专业总结

**效果**
- 从45分钟缩短到5分钟
- 语言更专业，条理更清晰
- 老板说："这月日报质量明显提高了"

### 踩过的4个坑（血泪教训）

| 坑 | 具体表现 | 正确做法 |
|----|----------|----------|
| 1. 问得太模糊 | "帮我写日报" | "帮我写今天的三车间日报，包含产量、能耗、故障和改进建议，语调要专业简洁" |
| 2. 直接复制AI输出 | AI有时会"一本正经地胡说八道" | 数字逻辑、专业术语必须核对一遍再发 |
| 3. 问一次就放弃 | 第一次问得不对，AI回答不好 | 换问法、分步骤问，好结果往往在第三、第四次 |
| 4. 指望AI替你做决定 | 让AI拍板安全生产、成本核算 | AI给参考、帮提效，最终拍板还是你自己 |

### 我的AI工具组合

| 场景 | 工具 | 用法 |
|------|------|------|
| 生产日报、会议纪要 | **OpenClaw（主力）** | 飞书里直接用，随时问 |
| 行业数据查询 | Tushare / AKShare | 查行情、跑数据 |
| 写方案、做PPT | Gamma / MindShow | 生成汇报材料 |
| 查资料、做分析 | Kimi / 智谱AI | 喂文档、读报告 |
