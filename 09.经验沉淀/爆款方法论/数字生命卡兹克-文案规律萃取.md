### 核心规律

**最佳甜蜜点：1200-2000字（5-9分钟）**

从数据来看，卡兹克阅读量 5万+ 的文章，绝大多数集中在这个区间：
- 太短（<800字）：信息密度不够，转发率低
- 太长（>3000字）：完读率下降，除非是系列深度文
- **1200-2000字是爆款密集区**，占5万+文章的约70%

## 📐 三、卡兹克文案结构规律

### 规律1：开篇黄金3行原则

卡兹克80%的高阅读文章，开篇前3行（约100字内）必须完成：
1. **抛出冲突/悬念**：让读者有继续看的欲望
2. **锚定场景**：让读者感知"这和我有关"
3. **给出信号**：暗示"看完你会得到什么"

**爆款开篇案例分析**：

```
「腾讯今年春节砸了10亿，就为了一件事——让更多人用元宝。
结果呢？
活动刚开始没几天，微信就以"诱导分享"为由，把元宝的推广链接给封了。」
→ 冲突（左手打右手）→ 悬念（为什么）→ 信号（大公司内部的荒诞）
```

```
「17岁高中生写了个神级Prompt，直接把Claude强化成了满血o1」
→ 人物冲突（17岁 vs Claude）→ 结果震撼（一击即中）
```

### 规律2：段落节奏「短-短-短-展」

卡兹克文章节奏非常鲜明：
- 连续3-5个短段（每段1-2行），制造呼吸感
- 然后1-2个展开段（4-6行），深入阐述
- 重复这个节奏，直到结尾

**禁止**：连续3段以上的长段落（读者会跳过）

### 规律3：金句密度

高阅读文章平均每400-500字有1句可传播金句：
- 「这已经不是量级的差距了，是生态位的差距。」
- 「用户习惯一旦形成，改变的成本极高。」
- 「大公司最大的竞争对手，往往是自己。」

金句特征：
- 8-20字之间
- 有对比或反转
- 可以单独截图传播
- 不需要上下文也能理解

### 规律4：结尾「行动号召」模式

卡兹克结尾几乎固定为2种模式：

**模式A（观点型）**：
```
总结核心观点（1-2句）
→ 对读者的启示（1-2句）
→ 转发/评论引导
```

**模式B（工具型）**：
```
总结使用价值（1-2句）
→ 社群/福利导流
→ 「在评论区扣XX，我拉你进去」
```

## 🔥 四、高阅读文章的文案共性特征

### 特征1：「我」视角贯穿全文

卡兹克10万+文章中，95%带有强烈第一人称：
- 「我觉得」「我发现」「我潜伏进了」「我总结了」
- 对比陈述式：「XX产品发布了，功能是XXX」（低点击）
- 我视角式：「我实测了XX，发现了一件让我震惊的事」（高点击）

### 特征2：数据具体化

高阅读文章的数据都非常具体：
- ✅「豆包月活2.3亿，元宝3286万，相差约7倍」
- ❌「豆包用户远多于元宝」
- ✅「除夕那天，豆包的AI互动总量破了19亿次，单分钟推理吞吐量最高达633亿tokens」
- ❌「春节期间豆包数据表现亮眼」

### 特征3：情绪词密度

高阅读文章每500字至少出现2-3个情绪词：
- 震撼类：「封神」「炸裂」「酷到」「惊叹」
- 共鸣类：「破防」「emo」「沉默了」「懵了」
- 紧迫类：「悄悄」「深夜」「刚刚」「突然」
- 反转类：「居然」「反而」「其实」「没想到」

### 特征4：对话感

卡兹克文章读起来像「和朋友聊天」，而非「写文章」：
- 高频使用：「你细品」「你会发现」「说真的」「其实吧」
- 结构上常用「然后呢？」「为什么？」「说得没错，但……」
- 转折词：「但」「不过」「只是」「只不过」

## ⏱️ 五、时长推荐决策树

```
选定选题类型
    ↓
是「时效热点」？
    ├── 是 → 推荐3-5分钟（700-1150字）
    └── 否 ↓
是「干货教程/专访人物」？
    ├── 是 → 推荐8-12分钟（1840-2760字）
    └── 否 ↓
是「观点/趋势/警示/反认知」？
    ├── 是 → 推荐6-9分钟（1380-2070字）
    └── 否 ↓
是「情感/娱乐/猎奇」？
    └── 是 → 推荐4-6分钟（920-1380字）
```

**原则**：宁可短一点，不要强行凑字数。卡兹克没有靠字数多取胜的爆款，但有不少短文因节奏快、金句密而爆。

## 📋 六、慎用写法（低阅读文章共性）

| 问题写法 | 对应数据低点 | 原因 |
|---------|-----------|------|
| 开篇直接介绍产品参数 | 「简单聊聊Google新发的视频模型Veo3.1」（3.5万） | 缺乏冲突和代入感 |
| 段落过长，连续超5行 | 低阅读文章普遍存在 | 手机阅读体验差 |
| 结论在文章最后才给 | 深度长文普遍 | 读者等不及 |
| 通篇客观陈述，无个人判断 | 「大模型盲测竞技场到底是个啥?」（2.7万） | 缺乏「我」视角 |
| 标题和内容不匹配（标题吹，内文平） | 部分2-3万文章 | 读者感觉被骗，不转发 |

## 🔑 七、Skill 调用规则

### 在 Step 5 生成文案前：

**Step A：确认时长**
1. 判断选题类型（参考上方决策树）
2. 给出推荐时长和字数
3. 等待用户确认或调整

**Step B：生成文案时的字数控制**
- 目标字数 = 用户确认时长 × 230字/分钟
- 允许浮动 ±10%
- 生成完成后，在文案末尾标注：「📝 本文约XXX字，预计朗读X分钟」
